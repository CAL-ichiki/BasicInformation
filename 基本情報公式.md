# 計算問題の公式集

## 基数変換

### ⭐⭐⭐ n進数 → 10進数
各桁の値 × n^(桁の位置) の合計

例: (1011)₂ = 1×2³ + 0×2² + 1×2¹ + 1×2⁰ = 8 + 0 + 2 + 1 = 11

### ⭐⭐⭐ 10進数 → n進数
nで割り続けて、余りを逆順に並べる

### ⭐⭐ 2進数 ⇔ 16進数
2進数4桁 = 16進数1桁
- (1111)₂ = (F)₁₆
- (1010)₂ = (A)₁₆

### ⭐⭐⭐ 負の数の表現（2の補数）
1. 絶対値を2進数で表現
2. ビットを反転（0→1、1→0）
3. 1を加算

例: -5（8ビット）
1. 5 = 00000101
2. 反転 = 11111010
3. +1 = 11111011

## CPU性能

### ⭐⭐⭐ MIPS（Million Instructions Per Second）
```
MIPS = クロック周波数（MHz） ÷ CPI
または
MIPS = 命令実行回数 ÷ 実行時間（秒） ÷ 1,000,000
```

### ⭐⭐ CPI（Cycles Per Instruction）
```
CPI = クロックサイクル数 ÷ 命令数
```

### ⭐⭐⭐ 実行時間
```
実行時間 = 命令数 × CPI ÷ クロック周波数
または
実行時間 = 命令数 ÷ MIPS ÷ 1,000,000
```

### ⭐⭐ プロセッサの性能比
```
性能比 = (クロック周波数A ÷ CPIA) ÷ (クロック周波数B ÷ CPIB)
```

### ⭐⭐ パイプライン処理の実行時間
```
実行時間 = (パイプライン段数 + 命令数 - 1) × 1クロックの時間
```

## メモリ

### ⭐⭐⭐ 実効アクセス時間
```
実効アクセス時間 = ヒット率 × キャッシュアクセス時間 + (1 - ヒット率) × 主記憶アクセス時間
```

例: キャッシュ10ns、主記憶100ns、ヒット率90%
```
= 0.9 × 10 + 0.1 × 100 = 9 + 10 = 19ns
```

### ⭐ メモリインタリーブの性能向上
```
性能向上率 = バンク数（理想的な場合）
```

### ⭐⭐ アクセス時間の計算
```
アクセス時間 = シーク時間 + 回転待ち時間 + データ転送時間
```

### ⭐⭐⭐ 平均回転待ち時間
```
平均回転待ち時間 = 1回転時間 ÷ 2
```

例: 7200rpm（1分間に7200回転）
```
1回転時間 = 60秒 ÷ 7200 = 0.00833秒 = 8.33ms
平均回転待ち時間 = 8.33 ÷ 2 = 4.17ms
```

## 稼働率

### ⭐⭐⭐ 直列システム
```
稼働率 = A × B × C × ...
```

例: 0.9 × 0.9 = 0.81（81%）

### ⭐⭐⭐ 並列システム（冗長化）
```
稼働率 = 1 - (1 - A) × (1 - B) × ...
```

例: 2台並列（各0.9）
```
= 1 - (1 - 0.9) × (1 - 0.9) = 1 - 0.1 × 0.1 = 1 - 0.01 = 0.99（99%）
```

### ⭐⭐ MTBF（Mean Time Between Failures：平均故障間隔）
```
MTBF = 総稼働時間 ÷ 故障回数
```

### ⭐⭐ MTTR（Mean Time To Repair：平均修理時間）
```
MTTR = 総修理時間 ÷ 故障回数
```

### ⭐⭐⭐ 稼働率の計算
```
稼働率 = MTBF ÷ (MTBF + MTTR)
```

例: MTBF=100時間、MTTR=5時間
```
稼働率 = 100 ÷ (100 + 5) = 100 ÷ 105 = 0.952（95.2%）
```

## ネットワーク

### ⭐⭐⭐ 伝送時間
```
伝送時間 = データ量（ビット） ÷ 回線速度（bps）
```

例: 1MBのファイルを10Mbpsで転送
```
1MB = 1 × 1024 × 1024 × 8 = 8,388,608ビット
伝送時間 = 8,388,608 ÷ 10,000,000 = 0.839秒
```

### ⭐⭐ 実効転送速度
```
実効転送速度 = データ量 ÷ (データ量 + オーバーヘッド) × 回線速度
```

### ⭐⭐⭐ IPアドレスのホスト数
```
ホスト数 = 2^(ホスト部のビット数) - 2
```
※ネットワークアドレスとブロードキャストアドレスを除く

例: /24（サブネットマスク255.255.255.0）
```
ホスト部 = 32 - 24 = 8ビット
ホスト数 = 2^8 - 2 = 256 - 2 = 254
```

### ⭐⭐ サブネット分割
```
サブネット数 = 2^(借用ビット数)
```

## 待ち行列理論

### ⭐⭐ 平均待ち時間（M/M/1モデル）
```
平均待ち時間 = ρ ÷ (μ - λ)
```
- λ: 到着率（単位時間あたりの到着数）
- μ: サービス率（単位時間あたりの処理数）
- ρ: 利用率 = λ ÷ μ

### ⭐⭐ 平均応答時間
```
平均応答時間 = 平均待ち時間 + 平均サービス時間
```

### ⭐⭐ 平均滞在数（Little's Law）
```
平均滞在数 = 到着率 × 平均応答時間
```

## プロジェクト管理

### アローダイアグラム

#### ⭐⭐ 最早開始時刻（Earliest Start Time）
各経路の最大値を採用（前方計算）

#### ⭐⭐ 最遅開始時刻（Latest Start Time）
各経路の最小値を採用（後方計算）

#### ⭐⭐⭐ 余裕時間（Slack Time）
```
余裕時間 = 最遅開始時刻 - 最早開始時刻
```

#### ⭐⭐⭐ クリティカルパス
余裕時間が0の経路

### ⭐⭐⭐ 工数計算
```
工数（人月） = 作業量 ÷ 生産性
期間（月） = 工数 ÷ 人数
```

例: 100人月の作業を10人で実施
```
期間 = 100 ÷ 10 = 10ヶ月
```

## 財務・会計

### ⭐⭐⭐ 損益分岐点売上高
```
損益分岐点売上高 = 固定費 ÷ (1 - 変動費率)
または
損益分岐点売上高 = 固定費 ÷ 限界利益率
```

```
変動費率 = 変動費 ÷ 売上高
限界利益率 = 1 - 変動費率 = 限界利益 ÷ 売上高
```

例: 固定費1000万円、変動費率60%
```
損益分岐点 = 1000 ÷ (1 - 0.6) = 1000 ÷ 0.4 = 2500万円
```

### ⭐⭐⭐ ROI（投資利益率）
```
ROI = 利益 ÷ 投資額 × 100（%）
```

例: 投資額1000万円、年間利益200万円
```
ROI = 200 ÷ 1000 × 100 = 20%
```

### ⭐⭐ 投資回収期間
```
投資回収期間 = 投資額 ÷ 年間利益
```

### ⭐ 正味現在価値（NPV）
```
NPV = Σ(各年のキャッシュフロー ÷ (1 + 割引率)^n) - 初期投資額
```

### ⭐⭐ 減価償却費（定額法）
```
年間減価償却費 = (取得原価 - 残存価額) ÷ 耐用年数
```

例: 取得原価1000万円、残存価額100万円、耐用年数5年
```
年間減価償却費 = (1000 - 100) ÷ 5 = 180万円
```

## 確率・統計

### ⭐⭐⭐ 期待値
```
期待値 = Σ(値 × 確率)
```

例: 当たり1000円（確率10%）、はずれ0円（確率90%）
```
期待値 = 1000 × 0.1 + 0 × 0.9 = 100円
```

### ⭐⭐ 標準偏差
```
分散 = Σ((各データ - 平均)^2) ÷ データ数
標準偏差 = √分散
```

### ⭐ 相関係数
```
-1 ≦ r ≦ 1
```
- r = 1: 完全な正の相関
- r = 0: 無相関
- r = -1: 完全な負の相関

## 論理演算

### ⭐⭐ AND（論理積）
```
0 AND 0 = 0
0 AND 1 = 0
1 AND 0 = 0
1 AND 1 = 1
```

### ⭐⭐ OR（論理和）
```
0 OR 0 = 0
0 OR 1 = 1
1 OR 0 = 1
1 OR 1 = 1
```

### ⭐⭐ XOR（排他的論理和）
```
0 XOR 0 = 0
0 XOR 1 = 1
1 XOR 0 = 1
1 XOR 1 = 0
```

### ⭐⭐ NOT（否定）
```
NOT 0 = 1
NOT 1 = 0
```

## ビット演算

### ⭐⭐⭐ シフト演算
- **左シフト（n桁）**: 2^n倍（2のn乗倍）
- **右シフト（n桁）**: 2^n分の1（2のn乗分の1）

例: 00001100（12）を左に2ビットシフト
```
00110000（48） = 12 × 2^2 = 12 × 4 = 48
```

### ⭐ マスク処理
特定のビットを取り出す・変更する

## 組み合わせ・順列

### ⭐⭐ 順列（Permutation）
```
nPr = n! ÷ (n - r)!
```

例: 5人から3人を選んで並べる
```
5P3 = 5! ÷ 2! = 120 ÷ 2 = 60通り
```

### ⭐⭐⭐ 組み合わせ（Combination）
```
nCr = n! ÷ (r! × (n - r)!)
```

例: 5人から3人を選ぶ
```
5C3 = 5! ÷ (3! × 2!) = 120 ÷ (6 × 2) = 10通り
```

H = -Σ(pi × log₂pi)

## 符号化

### ⭐⭐ ハミング距離
異なるビットの数

例: 1010 と 1100
```
ハミング距離 = 2（2ビット目と3ビット目が異なる）
```

### ⭐⭐ 検出可能なビット誤り
```
検出可能 = ハミング距離 - 1
```

### ⭐⭐ 訂正可能なビット誤り
```
訂正可能 = (ハミング距離 - 1) ÷ 2（小数点以下切り捨て）
```

## 暗号化

### ⭐⭐⭐ 共通鍵暗号の鍵数
```
n人で通信する場合の鍵数 = nC2 = n × (n - 1) ÷ 2
```

例: 5人
```
5C2 = 5 × 4 ÷ 2 = 10個
```

### ⭐⭐⭐ 公開鍵暗号の鍵数
```
n人の場合 = n組（各人が公開鍵と秘密鍵のペアを1組ずつ）
```

## 計算問題の解き方のコツ

1. **単位に注意**: bit/Byte、K/M/G、秒/分/時間
2. **公式を暗記**: 頻出公式は確実に覚える
3. **概算で検算**: 答えが妥当か確認
4. **図を描く**: ネットワーク構成や稼働率は図で考える
5. **過去問で練習**: 出題パターンを把握

### よく使う単位変換
- 1Byte = 8bit
- 1KB = 1024Byte = 2^10 Byte
- 1MB = 1024KB = 2^20 Byte
- 1GB = 1024MB = 2^30 Byte
- 1秒 = 1000ミリ秒（ms）
- 1ミリ秒 = 1000マイクロ秒（μs）
- 1マイクロ秒 = 1000ナノ秒（ns）
